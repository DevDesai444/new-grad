Per-company entry shape (default for first-time-scraped company):
```python
{
    "last_attempt_utc": None,           # ISO-8601 UTC string or None on first encounter
    "last_success_utc": None,           # ISO-8601 UTC string or None until first success
    "status": "ok",                     # enum: "ok" | "blocked" | "schema-drift" | "error"
    "consecutive_failures": 0,          # int; reset to 0 on success; increment on any failure
}
```

D-04b status classification rules (mapping orchestrator outcome → enum):
- outcome == "ok"                → status = "ok",          last_success_utc = run_started_at, consecutive_failures = 0
- outcome == "blocked"           → consecutive_failures += 1; status = "blocked" IFF consecutive_failures >= 3 else status = "error"
- outcome.startswith("error: SchemaDrift")   → consecutive_failures += 1; status = "schema-drift"
- outcome == "no-adapter"        → consecutive_failures += 1; status = "error" (treat CFG-05 as a failure to scan from a health-tracking perspective)
- any other outcome (PlaywrightTimeout / MissingCredential / InvalidCredential / generic) → consecutive_failures += 1; status = "error"

For ALL non-"ok" outcomes: last_attempt_utc = run_started_at; last_success_utc UNCHANGED.
For "ok" outcome: BOTH last_attempt_utc AND last_success_utc = run_started_at.
</source_health_schema>

           entry["status"] = status
           entry["consecutive_failures"] = new_fail
