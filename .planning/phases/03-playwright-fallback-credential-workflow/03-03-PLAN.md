```python
# After page.goto succeeds, BEFORE attempting XHR-intercept or DOM-fallback,
# check for a login form. If detected → attempt login → continue to extract.
# If env vars missing → raise MissingCredential.
# If login form persists after submit → raise InvalidCredential.

def _detect_login_form(self, page) -> bool:
    """Heuristic: page has an <input type='password'> within a form.

    Returns True if a password input is present; False otherwise.
    """
    try:
        return page.locator("input[type='password']").count() > 0
    except Exception:
        return False

def _attempt_login(self, page, company: CompanyConfig, timeout_ms: int) -> None:
    """Detect login form → read SCRAPER_<COMPANY>_<KIND> env vars → fill + submit.

    SEC-02: secret naming convention `SCRAPER_<COMPANY_UPPERCASE>_<KIND>`.
    SEC-03 / Pitfall 17: NEVER log credential values; NEVER include in
    exception messages. Generic failure messages only.
    """
    company_upper = company.name.upper().replace("-", "_").replace(" ", "_")
    email_var = f"SCRAPER_{company_upper}_EMAIL"
    password_var = f"SCRAPER_{company_upper}_PASSWORD"
    email = os.environ.get(email_var)
    password = os.environ.get(password_var)
    if not email or not password:
        raise MissingCredential(
            f"Playwright {company.name}: missing env var "
            f"{email_var} or {password_var}"
        )

    # Fill the form. Use generic selectors; per-site overrides go in v2.
    try:
        page.fill("input[type='email'], input[name='email'], input[name='username']", email)
        page.fill("input[type='password']", password)
        # Find a submit button: button[type='submit'] or input[type='submit']
        page.click("button[type='submit'], input[type='submit'], button:has-text('Sign in'), button:has-text('Log in')")
    except Exception as e:
        # NEVER include exception attrs (may contain DOM text including emails).
        raise InvalidCredential(
            f"Playwright {company.name}: login form fill/submit failed ({type(e).__name__})"
        ) from None  # `from None` suppresses chained traceback per SEC-03

    # Wait briefly; check the form is gone. If still present → bad creds.
    page.wait_for_timeout(3000)  # 3s; bounded — auth pages typically advance fast
    if self._detect_login_form(page):
        raise InvalidCredential(
            f"Playwright {company.name}: login form still present after submit "
            "(wrong credentials, anti-bot challenge, or selector drift)"
        )
```

## Adding a Company

When the user says "add this URL: <URL>", follow this 5-step flow autonomously:

### Step 1: Try existing adapters

Call `get_adapter(CompanyConfig(name=<derived>, url=<URL>))`. If an adapter matches (Greenhouse, Lever, Ashby, SmartRecruiters, Workday, Apple) → append `<URL>` to `companies.txt`, commit `chore(companies): add <name>`, push. **Done.**

### Step 2: Resolve redirects

Call `resolve_url(<URL>)` (Phase 3 Plan 03-01 helper). If the resolved URL differs from the original, re-try `get_adapter(CompanyConfig(name=<derived>, url=<URL>, resolved_url=<resolved>))`. If now matches (typical CNAME→Workday case) → append the **resolved** URL (not the original — per D-03a) to `companies.txt`. Commit. **Done.**

### Step 3: Playwright catch-all

If no specific adapter matches even after resolution, `PlaywrightAdapter` will handle it (registered LAST in `ADAPTERS`). Before committing:

1. Append `<URL>` to `companies.txt`.
2. Run a one-shot verification: `python -c "from src.adapters.playwright_fallback import PlaywrightAdapter; from src.models import CompanyConfig; r = PlaywrightAdapter().fetch(CompanyConfig(name='<derived>', url='<URL>')); print(len(r))"` — should print at least 1.
3. If verification succeeds → commit. **Done.**
4. If verification fails → step 4.

### Step 4: Write a new adapter (rare)

Playwright catch-all couldn't extract postings cleanly — the site has a unique structure benefiting from a dedicated adapter.

1. Create `src/adapters/<name>.py` subclassing `Adapter` (mirror `src/adapters/apple.py` shape).
2. Append `<NameAdapter>` to `ADAPTERS` in `src/registry.py` — **does NOT modify any existing adapter file** (ADP-15 invariant).
3. Append `_normalize_<name>` to `src/normalizer.py` `_DISPATCH`.
4. Write fixture + happy-path + 5 error-path tests mirroring `tests/test_apple_adapter.py`.
5. `pytest tests/ -v` — all 300+ tests still pass.
6. Append URL to `companies.txt`. Commit. **Done.**

### Step 5: Credential branch (jumps in BEFORE step 1 if detected)

When pasting a new URL, first probe whether it needs login: `httpx.get(<URL>, follow_redirects=True, timeout=5)` and search response body for `<input type="password">` or `<form action="*login*">`. If found → credentialed site.

Then (per SEC-01 / SEC-02 / SEC-04):

1. **Inline-prompt the user in chat:**

2. **Store secrets** (the user pastes them in chat once; never echo back):

   gh secret set SCRAPER_<COMPANY_UPPERCASE>_EMAIL --repo DevDesai444/new-grad --body "<email>"
   gh secret set SCRAPER_<COMPANY_UPPERCASE>_PASSWORD --repo DevDesai444/new-grad --body "<password>"
   ```
3. **Verify by listing** (names ONLY, never values per SEC-04):

   - If returns ≥1 RawPosting → credentials work; continue to step 1.

   - If raises `MissingCredential` → env-var setup failed; re-run the `gh secret set` and shell-env steps.

**Out of scope for v1** (per CONTEXT.md `<deferred>`): 2FA, OAuth, magic-link auth. If the site requires any of these, document as unsupported and skip.

## Credential Naming Convention (SEC-06)

When the user adds a site that requires login (Phase 3 Plan 03-03), Claude CLI provisions credentials via `gh secret set` with the naming convention:

```
SCRAPER_<COMPANY_UPPERCASE>_<KIND>
```

Where:
- `<COMPANY_UPPERCASE>` is the company name UPPERCASE with hyphens / spaces replaced by underscores (e.g., `ACME-CORP` → `ACME_CORP`).
- `<KIND>` is one of: `EMAIL`, `USERNAME`, `PASSWORD`, `API_KEY`, `OAUTH_TOKEN` (v1 supports `EMAIL` + `PASSWORD` only — other kinds out of scope).

### Per-Adapter Secret Audit

| Adapter | Secrets Referenced | Notes |
|---------|---------------------|-------|
| greenhouse | (none) | Public boards API; no auth |
| lever | (none) | Public postings API; no auth |
| ashby | (none) | Public posting-API; no auth |
| smartrecruiters | (none) | Public companies/postings API; no auth |
| workday | (none) | Public CXS /jobs endpoint; no auth |
| apple | (none) | Public api/role/search; no auth |
| **playwright** | `SCRAPER_<COMPANY>_EMAIL` + `SCRAPER_<COMPANY>_PASSWORD` (per credentialed company) | Read at scrape-time via `os.environ`; never logged, never written to repo |

### Audit, Rotate, Delete via `gh`

```bash
# List all secrets (names only — never values):
gh secret list --repo DevDesai444/new-grad

# Rotate one secret (provide new value via --body):
gh secret set SCRAPER_ACME_PASSWORD --repo DevDesai444/new-grad --body "<new-password>"

# Delete one secret:
gh secret delete SCRAPER_ACME_PASSWORD --repo DevDesai444/new-grad
```

**SEC-04 reminder:** `gh secret list` shows only secret NAMES, never values. Credential values exist only in: (a) GitHub's encrypted secret storage, (b) the runtime environment of the Actions runner during a scan, and (c) the user's local shell when explicitly re-provisioning. They never appear in repo files, chat history, or workflow logs (per SEC-03).

    def _detect_login_form(page) -> bool:

        Tolerant: never raises (a malformed page should not abort the scrape).

        """Convert company name to SCRAPER_<NAME>_<KIND> prefix per SEC-02.

        Uppercase; hyphens and spaces become underscores.
        E.g., 'acme-corp' -> 'ACME_CORP'; 'Big Co Inc.' -> 'BIG_CO_INC.'
        """

    def _attempt_login(self, page, company: CompanyConfig) -> None:
        """Detect login form → read SCRAPER_<COMPANY>_<KIND> env vars → fill + submit.

        Per CONTEXT.md D-02 + D-02a (eager prompt + separate per-kind secrets)
        and D-02c + SEC-03 + Pitfall 17 (structural ban on credential-value
        logging).

        Raises:
            MissingCredential: either email or password env var is unset.
            InvalidCredential: login form persists after submit + 3s wait,
                               OR a fill/click selector failure (which would
                               typically mean the site changed its login UI).

        prefix = self._company_to_secret_prefix(company.name)
        email_var = f"SCRAPER_{prefix}_EMAIL"
        password_var = f"SCRAPER_{prefix}_PASSWORD"
        email = os.environ.get(email_var)
        password = os.environ.get(password_var)
        if not email or not password:

                f"Playwright {company.name}: missing env var "

            )
            page.fill("input[type='password']", password)
            page.click(
