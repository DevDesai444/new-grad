    ```python
    """URL redirect resolver (CONTEXT.md D-01b — Phase 3 Plan 03-01).

    Unblocks the ~18-of-31 CNAME→Workday URLs in the user's companies.txt
    that the WorkdayAdapter's strict `.myworkdayjobs.com` host regex would
    otherwise miss. Cheap HEAD pre-flight; falls back to streaming GET when
    a site rejects HEAD; returns the original URL on any error (graceful
    degradation — NEVER raises to caller per D-01b).
    """
    from __future__ import annotations

    import logging

    import httpx

    logger = logging.getLogger("scan")

    _USER_AGENT = (
        "new-grad-tracker/0.1 (+https://github.com/DevDesai444/new-grad)"
    )
    _DEFAULT_TIMEOUT_S = 5.0


    def resolve_url(url: str, timeout_s: float = _DEFAULT_TIMEOUT_S) -> str:
        """Follow HTTP redirects from `url` and return the final URL.

        Tries HEAD first (cheap). If the server returns 405/501 (HEAD not
        allowed), falls back to streaming GET. On ANY error (timeout, network
        failure, 5xx), returns the original `url` unchanged — adapter
        dispatch then proceeds with the unresolved URL and may fall through
        to Playwright catch-all (Plan 03-02).

        Args:
            url: The raw URL from companies.txt (Phase 1 CompanyConfig.url).
            timeout_s: Per-request timeout (default 5.0s — keep cheap).

        Returns:
            The final URL after following 301/302/303/307/308 redirects, or
            the original `url` on any error.
        """
        headers = {"User-Agent": _USER_AGENT}
        try:
            r = httpx.head(
                url,
                follow_redirects=True,
                timeout=timeout_s,
                headers=headers,
            )
            if r.status_code in (405, 501):
                # HEAD not allowed — fall back to streaming GET (don't pull body).
                with httpx.stream(
                    "GET",
                    url,
                    follow_redirects=True,
                    timeout=timeout_s,
                    headers=headers,
                ) as g:
                    return str(g.url)
            if 200 <= r.status_code < 400:
                return str(r.url)
            # 4xx (other than 405/501) or 5xx — return original.
            logger.info(
                "resolve_url: %s returned HTTP %d, returning original",
                url, r.status_code,
            )
            return url
        except (httpx.TimeoutException, httpx.HTTPError) as e:
            # Per CONTEXT.md D-01b + SEC-03/Pitfall 17: log class + URL only,
            # never exception attrs (could include request headers).
            logger.info(
                "resolve_url: %s failed (%s), returning original",
                url, type(e).__name__,
            )
            return url


    __all__ = ["resolve_url"]
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| companies.txt → resolve_url | Untrusted user-supplied URLs. Attacker could craft a redirect chain that points to internal infrastructure (SSRF) or a very-slow server (DoS via 5s timeout). |
| resolve_url → external HEAD/GET requests | Outbound HTTP traffic to attacker-controllable destinations. Response bodies are NEVER read (HEAD or streaming GET that closes immediately). |
| Workflow YAML → Chromium download | actions/cache pulls from GitHub's cache infrastructure (trusted). `playwright install` downloads from Microsoft's CDN (trusted). |
| .gitignore → trace-file commit-time check | Trace files contain raw HTTP headers (potential cookies / session tokens). gitignore is the structural defense. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | I (Information Disclosure) | resolve_url exception logging | mitigate | Log `type(e).__name__` + URL only — never `traceback.format_exc()` which could capture request headers. | T-03-01-02 | T (Tampering) | Attacker-controlled redirect chain in companies.txt | accept | companies.txt is committed by the user via Claude CLI; attacker would need repo-write access, which would compromise the whole repo regardless. The resolver is no worse than the existing httpx.get in adapters. Document in CLAUDE.md (Plan 03-03) that pasted URLs should be from trusted sources. |
